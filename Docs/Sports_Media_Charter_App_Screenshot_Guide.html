1. Login Screen

Overview

The Login screen is the entry point of the Sports Media Charter app. It features the company branding with the airplane logo and tagline "Private Aviation for Professional Sports" against a navy-blue background.

Key Features

  • Email & Password Authentication — Standard credential-based login with a mail icon and lock icon for visual clarity.
  • Forgot Password — Link to initiate the password recovery flow.
  • Continue with SSO — Single Sign-On option for enterprise organizations that use identity providers.
  • Request Access — Link for new users to navigate to the registration screen.
  • Legal Footer — Tappable links to the Terms of Service and Privacy Policy, ensuring legal compliance from the first touchpoint.
Authentication SSO Password Recovery

The Login screen provides secure access to the app via email/password or SSO, with links to registration, password recovery, and legal disclosures. It establishes the premium Sports Media Charter brand identity.

2. Create Account

Overview

The registration screen allows new users to create an account to join Sports Media Charter. It uses a clean, minimal layout with a back arrow for easy navigation.

3. Home Dashboard

Overview

The Home Dashboard is the main landing screen after login. It provides a comprehensive at-a-glance overview of all charter activity with quick access to every major feature.

Key Features

  • Status Cards — Three color-coded cards showing Active Requests (blue), Pending Quotes (gold), and Upcoming Flights (navy) counts.
  • Quick Actions — Four action tiles: New Request, View Quotes, My Trips, and Team management.
  • Upcoming Flight — Displays the next confirmed flight with route (DFW → LAX), date, time, passenger count, aircraft type, and status badge.
  • Recent Activity — Timeline of recent events including new quotes received, requests submitted, and booking confirmations with timestamps.
  • Need Assistance Banner — Quick access to 24/7 support.
  • Tab Navigation — Bottom bar with Home, Requests, Book (+), Trips, Profile, and profile/edit tabs.
Dashboard Quick Actions Activity Feed Flight Status

The Home Dashboard serves as command center for charter operations. Users can instantly see their active workload, take quick actions, monitor upcoming flights, and review recent activity — all without leaving the screen.

7. New Charter Request — One Way

Overview

The New Charter Request form with "One Way" trip type selected. This is the primary booking flow where users specify all flight details to receive quotes from charter operators.

Key Features

  • Trip Type Selector — Three options: One Way (selected), Round Trip, and Multi-Leg. Each adjusts the form accordingly.
  • Flight Details — Departure and Arrival airport fields with a swap button between them, plus Date and Time pickers.
  • Passengers — Stepper control to set the number of passengers (minimum 1).
  • Urgency Level — Three tiers: Standard (48–72 hours), Urgent (24–48 hours), Emergency (under 24 hours). Standard is selected by default with a green checkmark.
  • Special Requirements — Free-text field for baggage notes, catering requests, accessibility needs, etc.
  • Submit Button — Gold "Submit Request for Quotes" button with airplane icon. Footer note indicates quotes arrive within 2–4 hours.
Booking Flow One Way Urgency Levels

app/(tabs)/new-request.tsx

Summary

The One Way request form captures all essential flight details in a single scrollable form. The urgency level system ensures the broker team prioritizes time-sensitive requests appropriately.

8. New Charter Request — Round Trip

Overview

The New Charter Request form with "Round Trip" selected. This variant automatically generates two flight legs — outbound and return — each with independent airport, date, and time fields.

Key Features

  • Leg 1 (Outbound) — Departure airport, arrival airport, date, and time for the outbound flight.
  • Leg 2 (Return) — Separate section with its own departure/arrival airports (pre-populated as reverse of Leg 1), date, and time fields.
  • Shared Settings — Passenger count, urgency level, and special requirements apply to both legs.
Booking Flow Round Trip Multi-Leg

app/(tabs)/new-request.tsx

Summary

The Round Trip form automatically creates two linked flight legs, streamlining the most common booking pattern for sports teams traveling to away games and returning home.

9. New Charter Request — Multi-Leg

Overview

The New Charter Request form with "Multi-Leg" selected. This variant supports complex itineraries with multiple stops, ideal for road trips or tournament travel.

Key Features

  • Multiple Legs — Each leg has independent departure/arrival airports, date, and time. Starts with two legs by default.
  • Add Another Leg — Button to add additional flight legs to the itinerary.
  • Remove Leg — Red delete icon on each leg card to remove a specific leg.
  • Leg Numbering — Clearly labeled "Leg 1", "Leg 2", etc. with colored badges.

The Multi-Leg form handles complex travel schedules with unlimited stops. This is essential for sports teams on multi-city road trips or tournament circuits.

11. Airport Search — By Airport Code

Overview

Airport search results when filtering by "Airport Code" and typing "lax". The search is case-insensitive and returns matching results instantly.

Key Features

  • Active Filter Tab — "Airport Code" tab is highlighted in dark navy, indicating the active search mode.
  • Result Card — Shows the IATA code (LAX) prominently, followed by the ICAO code (KLAX), full airport name, and location.
  • Airplane Icon — Visual indicator for commercial airports.
  • Clear Button — X button in the search field to clear the current query.
IATA Code ICAO Code Instant Search

src/components/AirportSearch.tsx

Summary

Searching by airport code is the fastest method for users who know the three-letter IATA code. Results display both IATA and ICAO codes along with the full airport name and city.

12. Airport Search — By City

Overview

Airport search results when filtering by "City" and typing "denver". This mode searches by city name rather than airport code.

Key Features

  • City Filter Active — The "City" tab is highlighted, filtering results to match by city name.
  • Location Display — Shows "Denver, CO, USA" as the location context beneath the airport name.
  • Single Result — Only DEN (Denver International Airport) matches the "denver" query in the city filter.
City Search Location Context

13. Airport Search — By Airport Name

Overview

Airport search results when filtering by "Airport Name" and typing "van". This mode searches the full airport name, returning diverse global results.

Key Features

  • Airport Name Filter — Searches within the full name of airports worldwide.
  • Multiple Results — Shows Vancouver (YVR), Van Nuys (VNY), and Helsinki-Vantaa (HEL) — all containing "van" in their names.
  • Private Airport Badge — VNY (Van Nuys) is tagged with a purple "Private" badge, distinguishing it from commercial airports.
  • Different Icons — Commercial airports show an airplane icon; private airports show a building icon.
